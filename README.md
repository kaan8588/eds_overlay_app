# EDS Overlay (Assistant): Intelligent Traffic Monitoring and Safety Warning System

This project is a mobile platform solution developed for educational and academic research purposes by Tufan Kaan İsli, a student at Istanbul Technical University, Computer Engineering. The application aims to increase driver safety and improve traffic awareness using Electronic Detection System (EDS) data.

## Academic Context and Purpose

The main goal of the project is to construct an "intelligent assistant" structure using geographic information systems (GIS), real-time data processing, and mobile sensor fusion techniques. The study focuses on the following academic problem solutions:

- **Spatial Indexing:** Low-latency querying on large-scale coordinate data.
- **Vector Orientation Filtering:** Development of trigonometric models that detect threats only in the user's direction of movement.
- **Battery Optimization:** Optimizing background services in accordance with Android lifecycle rules (Doze mode, Foreground limits).

---

## Technical Architecture

The project consists of an Android application (Mobile Client) structured in line with modern software architecture principles. The data required during the development process was obtained externally and integrated into the system.

### 1. Data Acquisition (Scraping)
The speed camera location data used within the scope of the application was obtained using Selenium-based dynamic web scraping and HTTP request techniques from General Directorate of Security (EGM) open map services.

### 2. Data Enrichment and LLM Integration (GenAI)
Speed limit information missing or needing update in the raw data has been autonomously enriched using Artificial Intelligence (LLM):
- **Google Gemini API Integration:** Legal speed limits were extracted from coordinates and description texts (D-100, Highway, City Center, etc.) using the `gemini-3.1-flash-lite-preview` model.
- **Batch Processing:** Data was grouped to stay within quota limits and an `exponential backoff` strategy was applied for error tolerance.
- **Validation Logic:** Speed data generated by the LLM was verified by comparing it with the valid set of values in Turkish traffic legislation (`VALID_SPEED_LIMITS`).

### 3. Mobile Application Layer (Android & Kotlin)
The application hosts an engine customized for high-performance spatial calculations.

#### A. Spatial Calculation Engine
A two-stage filtering is applied to prevent false positive warnings (parallel roads, cameras in the opposite direction):
- **Haversine Formula:** Calculates the Great-circle distance between two points on the spherical surface.
- **Azimuth (Bearing) Analysis:** Analyzes the angular difference between the user's movement vector and the EDS camera's perspective. Cameras within the `θ < 25°` tolerance interval are classified as "threats".

#### B. Data Management and Persistence
- **Room Persistence Library:** EDS data is stored in a SQLite-based database.
- **Bounding Box Query:** For performance, a bounding box query (Latitude/Longitude range) is first made to narrow down target points, then expensive trigonometric calculations are run only on this set.

#### C. Background Service and UI (Overlay Service)
- **Foreground Service:** A low-memory priority service architecture that continues to run even when the application is closed.
- **Custom View Animation:** A hardware-accelerated animation engine optimized for ambient lighting and visual warnings with `GlowParticlesView` was used.

#### D. Feedback and Privacy Architecture (Privacy-First Feedback)
The system's crowdsourced data collection (missing/wrong radar reporting) infrastructure is designed with the principle of network isolation:
- **Internet Independence:** The application does not request `INTERNET` permission in any way. Communication is delegated to the operating system using `Intent.ACTION_SENDTO`, and data integrity is ensured through the user's own mail client.
- **Opt-in Data Processing:** GPS data required for error detection is obtained only with the user's explicit consent and using instantaneous `lastKnownLocation`. All location and hardware data are processed within the phone and are not taken out of the device without permission.
- **Dynamic Screen Adaptation:** Autonomous scrolling (auto-scroll) and responsive layout mechanisms have been added for small screen devices and keyboard (IME) interactions using the `WindowInsets` API.

---

## Mathematical Foundations

The distance calculation algorithm used in the system is as follows:

$$d = 2r \arcsin\left(\sqrt{\sin^2\left(\frac{\phi_2 - \phi_1}{2}\right) + \cos(\phi_1) \cos(\phi_2) \sin^2\left(\frac{\lambda_2 - \lambda_1}{2}\right)}\right)$$

Where $\phi$ is latitude, $\lambda$ is longitude, and $r$ is the Earth's radius (6,371 km).

# EDS Overlay (Muavin): Akıllı Trafik Denetleme ve Güvenlik Uyarı Sistemi

Bu proje, İstanbul Teknik Üniversitesi Bilgisayar Mühendisliği bölümü öğrencisi Tufan Kaan İsli tarafından eğitim ve akademik araştırma amaçlı geliştirilmiş bir mobil platform çözümüdür. Uygulama, Elektronik Denetleme Sistemleri (EDS) verilerini kullanarak sürücü güvenliğini artırmayı ve trafik bilincini geliştirmeyi hedeflemektedir.

## Akademik Bağlam ve Amaç

Projenin temel amacı, coğrafi bilgi sistemleri (GIS), gerçek zamanlı veri işleme ve mobil sensör füzyonu tekniklerini kullanarak bir "akıllı asistan" yapısı kurgulamaktır. Çalışma kapsamında aşağıdaki akademik problem çözümlerine odaklanılmıştır:

- **Mekansal İndeksleme:** Büyük ölçekli koordinat verileri üzerinde düşük gecikmeli sorgulama.
- **Vektörel Yönelim Filtreleme:** Kullanıcının sadece hareket yönündeki tehditleri algılayan trigonometrik modellerin geliştirilmesi.
- **Batarya Optimizasyonu:** Arka planda sürekli çalışan servislerin Android yaşam döngüsü kurallarına (Doze mode, Foreground limits) uygun şekilde optimize edilmesi.

---

## Teknik Mimari

Proje, modern yazılım mimarisi prensipleri doğrultusunda yapılandırılmış bir Android uygulamasından (Mobil İstemci) oluşmaktadır. Geliştirme sürecinde ihtiyaç duyulan veriler dışarıdan temin edilerek sisteme entegre edilmiştir.

### 1. Veri Edinimi (Scraping)
Uygulama kapsamında kullanılan hız kamerası lokasyon verileri, Emniyet Genel Müdürlüğü (EGM) açık harita servislerinden Selenium tabanlı dinamik web scraping ve HTTP request teknikleri kullanılarak elde edilmiştir.

### 2. Veri Zenginleştirme ve LLM Entegrasyonu (GenAI)
Ham veride eksik olan veya güncellenmesi gereken hız limiti bilgileri, yapay zeka (LLM) kullanılarak otonom bir şekilde zenginleştirilmiştir:
- **Google Gemini API Entegrasyonu:** `gemini-3.1-flash-lite-preview` modeli kullanılarak koordinat ve açıklama metinlerinden (D-100, TEM, Şehir içi vb.) yasal hız limitleri çıkarılmıştır.
- **Toplu İşlem (Batch Processing):** Kota sınırları içerisinde kalmak için veriler gruplandırılmış ve hata toleransı için `exponential backoff` stratejisi uygulanmıştır.
- **Validasyon Mantığı:** LLM tarafından üretilen hız verileri, Türkiye trafik mevzuatındaki geçerli değerler kümesiyle (`VALID_SPEED_LIMITS`) karşılaştırılarak doğrulanmıştır.

### 3. Mobil Uygulama Katmanı (Android & Kotlin)
Uygulama, yüksek performanslı mekansal hesaplamalar için özelleştirilmiş bir motor barındırır.

#### A. Mekansal Hesaplama Motoru (Spatial Engine)
Hatalı pozitif uyarıları (paralel yollar, ters istikametteki kameralar) engellemek için iki aşamalı bir filtreleme uygulanır:
- **Haversine Formülü:** Küresel yüzey üzerindeki iki nokta arasındaki büyük daire mesafesini (Great-circle distance) hesaplar.
- **Azimut (Rulman) Analizi:** Kullanıcının hareket vektörü ile EDS kamerasının bakış açısı arasındaki açısal farkı analiz eder. `θ < 25°` tolerans aralığındaki kameralar "tehdit" olarak nitelendirilir.

#### B. Veri Yönetimi ve Kalıcılık
- **Room Persistence Library:** EDS verileri SQLite tabanlı bir veritabanında saklanır.
- **Bounding Box Query:** Performans için önce bir sınırlayıcı kutu sorgusu (Latitude/Longitude range) yapılarak aday noktalar küçültülür, ardından maliyetli trigonometrik hesaplamalar sadece bu küme üzerinde çalıştırılır.

#### C. Arka Plan Servisi ve UI (Overlay Service)
- **Foreground Service:** Uygulama kapalıyken dahi çalışmaya devam eden, düşük bellek öncelikli bir servis mimarisi.
- **Custom View Animation:** `GlowParticlesView` ile ortam ışıklandırması ve görsel uyarılar için optimize edilmiş, donanım hızlandırmalı bir animasyon motoru kullanılmıştır.

#### D. Geri Bildirim ve Gizlilik Mimarisi (Privacy-First Feedback)
Sistemin kitle kaynaklı veri toplama (eksik/hatalı radar bildirimi) altyapısı, ağ izolasyonu (network isolation) prensibiyle tasarlanmıştır:
- **İnternet Bağımsızlığı:** Uygulama hiçbir şekilde `INTERNET` izni talep etmez. İletişim, `Intent.ACTION_SENDTO` kullanılarak işletim sistemine devredilir ve veri bütünlüğü kullanıcının kendi mail istemcisi üzerinden sağlanır.
- **Opt-in Veri İşleme:** Hata tespiti için gereken GPS verileri, yalnızca kullanıcının açık rızasıyla ve anlık `lastKnownLocation` kullanılarak elde edilir. Tüm konum ve donanım verileri telefon içerisinde işlenerek cihaz dışına izinsiz çıkarılmaz.
- **Dinamik Ekran Adaptasyonu:** `WindowInsets` API kullanılarak küçük ekranlı cihazlar ve klavye (IME) etkileşimleri için otonom kaydırma (auto-scroll) ve duyarlı (responsive) layout mekanizmaları eklenmiştir.
